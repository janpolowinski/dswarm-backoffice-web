How To Copy or Migrate Mappings

  • Select a data model (that should act as input data model) and a project (that should act as reference project, i.e., where the mappings and the output data model are taken from) and apply "Copy Mappings" or "Migrate Mappings"
  • note: the mapping inputs of the mappings (of the reference project) must be contained in the schema of the selected (input) data model for "Copy Mappings"
  • note: the mapping inputs of the mappings (of the reference project) must be contained somehow similar in the schema of the selected (input) data model "Copy Mappings"
  • note: the input data model is optional for "Copy Mappings"
  • both actions should result in a new project with copied or migrated mappings
  • please also have a look at the documentation for further details about these actions
